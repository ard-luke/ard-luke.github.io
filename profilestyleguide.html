Bootstrap Grid

General Grid

The entire content of the profile page is within a <div class="container">.

The #at-a-glance section is contained within its own <div class="col-12"> so that it's always full-width.

The #education, #professional-exp, and #personal-info sections are all contained within <div class="col-lg-8 col-sm-12"> so that they switch to full-width at the lg bootstrap breakpoint.

The #contact section is contained within <div class="col-lg-4 col-sm-12"> so that it switches to full-width at the lg bootstrap breakpoint.

Nested Grids

Most div's within a section have the class col-12 to ensure they remain full-width

The #contact section has several column classes to ensure some information switches from being inline with its subheader (when the section is full-width at smaller screen widths) to being below its subheader (when the section is a sidebar at larger screen widths):

<div class="card-content-category-sub col-4 col-lg-12">Personal (Preferred)</div>
<div class="card-content-category-info col-8 col-lg-12">cat@alum.northwestern.edu</div>
<div class="card-content-category-sub col-4 col-lg-12">Work</div>
<div class="card-content-category-info col-8 col-lg-12">cat.fields@abbot.com</div>

The #personal-info section has a similar nested bootstrap grid to achieve the same inline subhead/information layout: 

<div id="names-full" class="card-content-category-sub col-4">Full Name</div>
<div id="names-full" class="card-content-category-info col-8">Ms. Catrina Fields</div>

Spacing

All spacing uses multiples of rem for margins, padding, and typography.

Most elements that are related are separated by 1 rem, and elements that are unrelated are separated by 2 rem.
Below is a list of spacing styles for reference:

Sections/Cards

All sections/cards are separated by a bottom margin of 30px:
section.row{ margin-bottom: 30px; }

Sections/cards have a padded header div "card-top":
.card-top{ padding: 1rem 1.5rem 1rem 1.5rem; }

Content within sections/cards are padded:
.card-content{ width: 100%; padding: 1rem 1.5rem; }

Info Fields

Info field categories (with subheads like "Northwestern Education" and "Reunion Year") all have a bottom margin of 1 rem:
`.card-content-category:last-child *:last-child { margin-bottom: 0; }

Subheads

All subheads (".card-content-category-name") have a bottom margin of 0.3125 rem (5px):
.card-content-category-name{ margin-bottom: 0.3125rem; }

Logos/Icons and Indented Info

Info fields like "Northwestern Education" and "Life Events" have logos/icons that float left of the field's information.

Logos/Icons are contained within <div class="logo-wrapper"> tags, and have a fixed width, height, and right margin:

.logo-wrapper{ 
	width: 50px;
	height: 50px;
	margin-right: 1rem;
}

Information to the right of a logo/icon has a width equal to the width of its parent container, minus the sum of the logo/icon and its margin: .info-wrapper{ display: inline-block; width: calc(100% - 66px); }

example
Northwestern Degree(s)
McCormick School of Engineering
BS, Environmental Engineering
Class of 2008

You can use .card-content-narrow to achieve the same effect without a logo/icon.

Box-shadow

All content sections/cards have a box-shadow (except for the contact section):

The #at-a-glance section is the only part of the profile that is center-aligned.

The .at-a-glance-background-img is the background image at the top of the section:

Radio Buttons

Radio buttons float left of the information they are paired with by way of .logo-wrapper and .info-wrapper. See the spacing section for more info on how those work.

The custom radio buttons are styled by hiding the <input type="radio">, and styling <div class="check"></div> within the input's <label>.

Each <label> tag is a child of a <li> within an <ul>. Interactions are implemented by ul li:hover .check{}

example

Checkboxes

Checkboxes are styled by essentially the same process as radio buttons, except instead of <input type="radio">, and styling <div class="check"></div>, we use <input type="checkbox">, and styling <div class="check-box"></div>

Interactions are implemented by ul li:hover .check-box{}

Modal Windows

Modal windows are implemented with Bootstraps's built-in modal component.

Modal windows have a height of 100% and a max-height of 660px so that it has a lightbox effect on desktop, but fills the screen vertically on mobile devices: 

.modal-content {
	border-radius: 0;
	border: none;
	/* Bootstrap sets the size of the modal in the modal-dialog class, we need to inherit it */
	width:inherit;
	height: 100%;
	/* To center horizontally */
	margin: 0 auto;
}

.modal-dialog{
	overflow-y: initial !important;
	height: 100%;
	max-height: 660px;
}

Modals are vertically centered using parent divs .vertical-alignment-helper and .vertical-align-center: 

.vertical-alignment-helper {
	display:table;
	height: 100%;
	width: 100%;
}
.vertical-align-center {
	/* To center vertically */
	display: table-cell;
	vertical-align: middle;
}

The bootstrap .btn-primary class is used for both the save and delete buttons because I wanted to carry over the existing button hover animation from the OurNorthwestern directory.

Text Inputs and Dropdowns

Style

I tried making text input fields and dropdowns as close to the existing design for inputs on the OurNorthwestern directory. This means no border-radius, a 1px solid off-black border, and a #4e2a84 border when selected using the .form-control class.

NOTE: I did not mimic the input fields from the directory perfectly, but the closer it is the better for continuity’s sake.

Medium/Third Wide Inputs

To have an input (text or dropdown) span about 45% of its container's width use .med-wide

To have an input (text or dropdown) span about 30% of its container's width use .third-wide

example

Input Error State

If someone tries to save an invalid input for any given information field, an error state would take effect for that input. 

label.error-state::after {
	content: ' *';
	color: #ff0000;
}

input.form-control.error-state {
    border: 1px solid;
    border-color: #ff0000;
    box-shadow: 0px 0px 3px 0px rgba(255,0,0,0.3);
    margin-bottom: 0;
}

input.form-control.error-state ~ .error-state-msg {
	color: #ff0000;
	font-size: 0.9rem;
}
